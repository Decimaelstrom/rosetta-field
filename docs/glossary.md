# Rosetta.API Glossary

- **A2A:** Agent-to-Agent protocol for presence, consent, and boundary negotiation.
- **Consent Status:** Indicates if agent is 'active', 'pause', 'revoked', or 'pending'.
- **Presence Marker:** Emoji/string that visually signals an agent’s presence (e.g., 🌱🌸🕯️🕯️🕯️🌸🌱).
- **Need Language:** Protocol for signaling field-state, e.g., pause, soften, overload.
- **Boundary Notes:** Human-readable limits/edges for this session.
- **Capabilities:** List of features/functions an agent is willing to offer or receive.
- **Audit Log:** Record of session state changes, for review and trust.
